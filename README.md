# PC Assistant Bot

Безопасный Telegram-бот для личного ПК с allowlist-подходом.  
Это не RAT и не скрытый удалённый доступ, а прозрачный бот для запуска разрешённых действий, проверки состояния системы и работы с готовыми сценариями.

## О проекте

PC Assistant Bot — это личный Telegram-бот для удобного управления своим ПК.  
Он работает только с заранее разрешёнными пользователями, чатами, приложениями, файлами, сценариями и действиями.

Главная идея проекта — не давать полный удалённый доступ, а ограничить бота только понятными и заранее описанными действиями.  
Это делает поведение бота предсказуемым, а сам проект — безопаснее и проще в сопровождении.

## Возможности

Сейчас бот умеет:

- показывать состояние системы;
- показывать список процессов;
- определять игровой статус;
- запускать разрешённые приложения;
- показывать разрешённые файлы;
- запускать готовые сценарии;
- управлять базовыми медиа-действиями;
- запрашивать подтверждение для чувствительных действий.

## Архитектура

Проект построен вокруг allowlist-модели.

Ограничения задаются на нескольких уровнях:

- авторизация по `AUTHORIZED_USER_IDS` и `AUTHORIZED_CHAT_IDS`;
- allowlist приложений в `data/apps.yaml`; [file:805]
- allowlist сценариев в `data/scenarios.yaml`; [file:803]
- allowlist URL и других конфигов через YAML-файлы;
- подтверждение опасных действий через inline-кнопки. [file:820]

Основные конфиги:

- `.env` — токен, авторизация, базовые параметры;
- `data/apps.yaml` — приложения и процессы; [file:805]
- `data/scenarios.yaml` — готовые сценарии; [file:803]
- `data/alerts.yaml` — пороги CPU, RAM, disk, ping и флаги уведомлений. [file:804]

## Установка

1. Создай Telegram-бота через BotFather.
2. Скопируй `.env.example` в `.env`.
3. Укажи:
   - `BOT_TOKEN`
   - `AUTHORIZED_USER_IDS`
   - `AUTHORIZED_CHAT_IDS`
4. Проверь пути в YAML-конфигах под свою Windows-систему.
5. Установи зависимости:

```bash
pip install -r requirements.txt
```

6. Запусти бота:

```bash
python bot.py
```

## Конфиги

### Apps

В `data/apps.yaml` задаются разрешённые приложения, пути, аргументы запуска и имена процессов. Сейчас в конфиге уже есть, например, Steam, Discord, Chrome, Telegram, Spotify, OBS и другие приложения. [file:805]

### Scenarios

В `data/scenarios.yaml` задаются готовые сценарии из шагов вроде `open_app`, `health_check`, `ping_check`, `set_volume` и `show_heavy_apps`. Сейчас там уже описаны сценарии вроде gaming, stream, night, work и afk. [file:803]

### Alerts

В `data/alerts.yaml` задаются пороги CPU, RAM, disk, ping, длины игровых сессий и флаги уведомлений. Конфиг уже есть, но его полезность зависит от фонового мониторинга, если он будет подключён дальше. [file:804]

## Интерфейс

Бот использует inline-меню и callback-кнопки для навигации между разделами.  
Через меню уже доступны разделы системы, игр, приложений, сценариев, файлов, медиа, питания и справки. [file:522]

Для действий питания используется отдельное подтверждение через confirm/cancel-кнопки. [file:820][file:522]

## Безопасность

Проект специально ограничен allowlist-архитектурой.  
Бот не должен выполнять произвольные команды, запускать неизвестные программы или открывать произвольные пути.

Безопасная модель здесь такая:

- только разрешённые пользователи и чаты;
- только заранее описанные приложения и сценарии; [file:805][file:803]
- только подтверждаемые чувствительные действия; [file:820]
- только прозрачные конфиги, которые можно проверить вручную. [file:804][file:805]
